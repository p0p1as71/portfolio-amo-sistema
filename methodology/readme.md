# Metodología de Trabajo con IA

Esta sección describe la **metodología operativa y mental**
utilizada para diseñar y evolucionar el sistema A.M.O.

No se trata de una metodología de desarrollo de software tradicional,
sino de un enfoque para **trabajar con sistemas de IA complejos**
sin perder control ni coherencia.

Aquí se documentan principios como:
- Separación entre diseño, ejecución y razonamiento
- Uso de contextos canónicos persistentes
- Chats desechables y documentación duradera
- Prompts mínimos, explícitos y restrictivos

La metodología asume que:
- La IA razona, pero **no gobierna**
- El control estructural reduce errores más que la iteración informal
- La claridad documental es parte del sistema

Esta carpeta responde a la pregunta:

> **Cómo se trabaja con A.M.O. sin degradar su arquitectura.**
