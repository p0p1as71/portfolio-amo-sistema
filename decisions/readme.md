# Decisiones Arquitectónicas (ADRs)

Esta sección recoge las **decisiones arquitectónicas clave** del sistema A.M.O.
en forma de *Architectural Decision Records* (ADRs).

Cada ADR documenta:
- El **problema real** que se intenta resolver
- Las **alternativas consideradas**
- La **decisión tomada**
- Las **consecuencias y trade-offs** asumidos

El objetivo no es justificar decisiones a posteriori,
sino **hacer explícito el razonamiento arquitectónico**
y permitir su auditoría futura.

Las decisiones aquí reflejadas priorizan:
- Gobernanza sobre inteligencia
- Control sobre flexibilidad
- Trazabilidad sobre optimización prematura

Esta carpeta responde a la pregunta:

> **Por qué el sistema es así y no de otra manera.**
