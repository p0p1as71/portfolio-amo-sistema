# Arquitectura de A.M.O.

Esta sección documenta la **arquitectura conceptual** del sistema A.M.O.

Aquí se describen:
- Los **módulos principales** del sistema
- Sus **responsabilidades explícitas**
- Los **límites funcionales** entre ellos
- Los **flujos de interacción permitidos**

La arquitectura de A.M.O. no se organiza por capacidades técnicas,
sino por **responsabilidades y autoridad**.

No se detallan implementaciones ni tecnologías concretas.
El foco está en **cómo debe estructurarse un sistema de IA gobernado**
para evitar solapamientos, decisiones no autorizadas y deriva semántica.

Los documentos de esta carpeta responden a la pregunta:

> **Qué es cada parte del sistema y qué NO le está permitido hacer.**
